¿Cómo diseñar una página web de forma sencilla que transmita una identidad de marca unificada, mejore la experiencia de usuario y además se pueda adaptar a los distintos centros del CSIC y sus particularidades?

Creando una plantilla modular que transmita una identidad digital coherente, proporcione una navegación optimizada mediante una arquitectura de información clara, y que permita una adaptabilidad a las características específicas de cada centro.

01. Descubrir, entender y empatizar

Comprendiendo el desafío y los objetivos de la investigación

  • Identificar requisitos y expectativas mediante entrevistas con stakeholders clave (Product Owner y responsables de gobernanza web).
  • Análisis de referencias web para identificar perfiles de usuario, servicios y buenas prácticas.
  • Investigación de necesidades y frustraciones de los principales usuarios de las webs del CSIC.

Hallazgos de la investigación

  • Falta de coherencia visual y funcional que debilita la identidad del CSIC.
  • El diseño debe alinearse con las webs existentes y cumplir los manuales de identidad institucional y de la AGE.
  • Navegación confusa, ausencia de búsqueda avanzada y dificultades para acceder a contactos clave.
  • El contenido está desactualizado y poco estructurado, afectando su claridad y relevancia.

02. Definir el desafío

Target audience

Tras la fase de research se definió centrar el MVP en el perfil de user persona principal: personal científico externo. El resto del perfiles de usuario (personal interno, empresas, medios de comunicación y sociedad) serán abordados en futuras iteraciones.

User journey

Problem statement

Los usuarios clave de las webs de los centros del CSIC, especialmente el personal científico externo, enfrentan dificultades para acceder a información esencial, como la consulta de proyectos o detalles de contacto y cómo establecerlo. El objetivo del presente proyecto es crear una plantilla web facilmente adaptable que fortalezca la identidad del CSIC y facilite el acceso a información y opciones de contacto, mejorando así la experiencia del principal usuario definido en para este MVP.

03. Idear la solución

How Might We

El primer paso para idear un MVP que respondiera a los obejtivos planteados tras la fase de investigación comenzó por una exploración de cómo podríamos solventar los problemas y desafíos detectados. Para ello se plantearon los siguientes “How Might We”:

Ver How might we

Priorización

Se utilizó una combinación de matriz de usabilidad y viabilidad junto con el método MoSCoW para priorizar soluciones, equilibrando las necesidades del user persona con las limitaciones técnicas y los requisitos del proyecto. El MVP incluirá: navegación simplificada, estructura clara para acceder a información, herramientas de contacto mejoradas y una plantilla adaptable que refuerce la identidad del CSIC.

Ver MoSCoW

Information Architecture

La página de inicio se definió como el núcleo de navegación, conectando directamente a secciones clave como departamentos, proyectos, publicaciones, servicios y contacto, asegurando un acceso rápido en pocos clics. La jerarquía de la sección de investigación se estructuró de forma familiar para los usuarios, basándome en patrones comunes de webs del CSIC. Además, el sitemap consideró conexiones entre páginas para facilitar la consulta de información relacionada y el contacto con los centros.

Information Architecture

User flow

El principal flujo de usuario se centró en facilitar el acceso a información estructurada sobre proyectos (Departamento > Grupos > Proyectos). En caso de no encontrar el contenido necesario, el usuario debía poder contactar fácilmente con el personal investigador o del centro, integrando este aspecto en el diseño del flujo de interacción.

Ver user flows

Enfoque MVP para resolver el Problem Statement

Durante esta fase, se definieron las siguientes ideas clave para el desarrollo de un MVP que respondiera a las necesidades del personal científico externo y los objetivos del proyecto:

  • Plantilla modular y flexible que refuerce la identidad visual del CSIC y aproveche las capacidades de WordPress y Divi.
  • Navegación simplificada con un menú claro y familiar para facilitar el acceso al contenido.
  • Accesibilidad y búsqueda eficiente mediante accesos rápidos y filtros avanzados para localizar información clave.
  • Contacto directo a través de un botón visible que facilite el acceso a formularios y detalles de contacto del personal clave.

04. Diseñar con propósito

Wireframes

Se crearon wireframes de baja fidelidad para 8 pantallas específicas, diseñadas para facilitar el acceso a recursos clave como proyectos, publicaciones y contacto. Estos ayudaron a detallar cada paso del usuario, establecer jerarquías claras de contenido y estructurar una navegación modular e intuitiva que mejorara la experiencia de consulta de información.

Ver wireframes

Visual guide and design system

Se creó un sistema de diseño basado en un enfoque atómico para definir los aspectos visuales y funcionales de la interfaz. Esto incluyó decisiones estratégicas sobre tipografía, paleta de colores y componentes reutilizables como botones y elementos de navegación. El sistema asegura una identidad digital consistente, adaptable a distintos centros del CSIC y alineada con los requisitos de los stakeholders, sirviendo como base para desarrollar el prototipo.

05. Prototipar la experiencia

Prototipo en alta

Se desarrolló un prototipo enfocado en representar el flujo del usuario a través de las pantallas clave. El menú principal se diseñó como elemento central de la navegación, mientras que la búsqueda de proyectos incluyó tabs con filtros por departamento. Además, se creó un formulario de contacto que permite dirigir solicitudes a departamentos específicos, facilitando la comunicación con el personal del centro.

06. Testear & Validar

Usability test plan & feedback collection

Se realizaron pruebas de usabilidad con 4 investigadoras externas y 3 stakeholders, evaluando tareas clave como búsqueda de proyectos, consulta de publicaciones y contacto con responsables. Los resultados confirmaron buena accesibilidad y claridad, pero identificaron áreas de mejora, como agregar un menú fijo, un botón flotante para volver al principio de la página y una forma más clara y organizada de presentar los datos de contacto. Estos hallazgos validaron las fortalezas del prototipo y ayudaron a priorizar ajustes para optimizar la experiencia del usuario.

Ver feedback collection

Learnings y nexts steps

  • Los desafíos incluyeron equilibrar las necesidades de los usuarios con los requisitos del Product Owner y la governance, enfrentando problemas de navegación confusa, búsqueda poco eficaz y diseño inconsistente.
  • Diseñar priorizando claridad y accesibilidad resultó esencial, y unificar la identidad visual del CSIC es viable sin comprometer la flexibilidad de los centros.
  • Los próximos pasos consisten en optimizar la navegación con un menú fijo y botones de retorno, implementar filtros avanzados y búsquedas combinadas, añadir preguntas frecuentes y formularios específicos, y validar las mejoras con iteraciones continuas.
